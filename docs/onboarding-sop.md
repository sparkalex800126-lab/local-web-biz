# 客戶 Onboarding SOP — 建站交付流程

> 客戶付款（訂金）後的完整執行流程。每一步都有明確的負責方與時間。

---

## 流程總覽

```
簽約付訂 → 收集素材 → 正式建站 → 初稿審閱 → 修改調整 → 域名設定 → 正式上線 → 年度維護
   ↓          ↓          ↓          ↓          ↓          ↓          ↓          ↓
 Day 0     Day 1-7    Day 8-14   Day 15-21  Day 22-28  Day 29-30  Day 31     持續
```

---

## Step 1：收集素材（Day 1–7）

**負責方：** 客戶（甲方）  
**目標：** 取得建站所需的全部素材

### ✅ Checklist

- [ ] 發送「素材收集表」給客戶（見下方模板）
- [ ] 確認客戶收到並了解需提供的內容
- [ ] 提醒客戶 7 天內繳回
- [ ] 收到素材後確認完整性
- [ ] 素材不齊全 → 列出缺少項目通知客戶補齊

## Step 2：正式建站（Day 8–14）

**負責方：** PageReady TW（乙方）  
**目標：** 完成網站初稿

### ✅ Checklist

- [ ] 確認素材已全部到齊
- [ ] 基於 demo 樣板選擇最適合的版型
- [ ] 替換客戶素材（Logo、照片、文案）
- [ ] 調整品牌色系與風格
- [ ] 嵌入 Google Maps
- [ ] 設定社群連結（LINE / FB / IG）
- [ ] 設定聯絡表單 / LINE 按鈕（方案 B/C）
- [ ] 設定預約功能（方案 B/C）
- [ ] 建立子頁面（方案 C）
- [ ] 基礎 SEO 設定（方案 C）
- [ ] RWD 測試（手機 / 平板 / 桌機）
- [ ] 部署至預覽網址
- [ ] 內部品質檢查（連結、錯字、圖片載入）

## Step 3：初稿給客戶審閱（Day 15–21）

**負責方：** 雙方  
**目標：** 客戶確認初稿方向正確

### ✅ Checklist

- [ ] 將預覽網址傳送給客戶
- [ ] 附上「審閱重點說明」（看什麼、怎麼回饋）
- [ ] 提醒客戶 7 天內回覆
- [ ] 客戶回覆 → 進入 Step 4
- [ ] 客戶未回覆 → 第 5 天提醒一次
- [ ] 超過 7 天未回覆 → 再次提醒並說明時程影響

### 📝 審閱重點說明（傳給客戶）

> 請您看看以下幾個重點，有任何想調整的地方請一次告訴我們：
> 1. 整體風格和顏色喜不喜歡？
> 2. 照片有沒有想換的？
> 3. 文字內容有沒有要修改的？
> 4. 功能操作起來順不順？（表單、按鈕等）
> 5. 其他任何想法都歡迎提出！

## Step 4：修改（Day 22–28）

**負責方：** PageReady TW（乙方）  
**目標：** 根據客戶回饋調整至定稿

### ✅ Checklist

- [ ] 彙整客戶修改意見
- [ ] 判斷修改項目是否在合約範圍內
- [ ] 範圍外項目 → 通知客戶並提供加購報價
- [ ] 執行第 1 次修改 → 通知客戶確認
- [ ] 執行第 2 次修改 → 通知客戶確認（如需要）
- [ ] 執行第 3 次修改 → 通知客戶確認（如需要）
- [ ] 修改次數用完 → 告知客戶後續修改費用（$1,500/次）
- [ ] 客戶確認定稿 ✅

---

## Step 5：域名設定（Day 29–30）

**負責方：** PageReady TW（乙方）+ 客戶配合  
**目標：** 設定正式域名指向網站

### 情境 A：客戶已有域名

- [ ] 請客戶提供域名管理後台帳號，或協助設定 DNS
- [ ] 設定 DNS A Record / CNAME 指向主機
- [ ] 等待 DNS 生效（通常 1-24 小時）
- [ ] 設定 SSL 憑證

### 情境 B：代客戶購買域名

- [ ] 與客戶確認想要的域名名稱
- [ ] 查詢域名是否可註冊
- [ ] 告知域名年費並取得客戶同意
- [ ] 註冊域名（以客戶名義或代管）
- [ ] 設定 DNS 指向
- [ ] 設定 SSL 憑證

---

## Step 6：正式上線 + 交付（Day 31）

**負責方：** PageReady TW（乙方）  
**前提：** 客戶確認定稿 ＋ 尾款已入帳

### ✅ Checklist

- [ ] 確認尾款已收到
- [ ] 將網站從預覽環境搬到正式環境
- [ ] 綁定正式域名
- [ ] 最終檢查：所有頁面、連結、表單正常運作
- [ ] 提交 Google Search Console（方案 C）
- [ ] 交付以下項目給客戶：
  - [ ] 網站管理帳號密碼（如適用）
  - [ ] 網站原始碼（如客戶要求）
  - [ ] 簡易操作說明文件
  - [ ] DNS / 域名管理資訊
- [ ] 通知客戶網站已正式上線 🎉
- [ ] 請客戶確認驗收（書面或 LINE 回覆「確認」即可）

---

## Step 7：年度維護開始

**負責方：** PageReady TW（乙方）  
**起算日：** 網站正式上線日

### ✅ Checklist

- [ ] 確認客戶是否加購年度維護方案（$4,800/年）
- [ ] 如已加購 → 說明維護內容（詳見 `maintenance-plan.md`）
- [ ] 設定 uptime 監控
- [ ] 建立客戶維護紀錄表
- [ ] 排定每月主動聯繫時間（確認有無更新需求）
- [ ] 如未加購 → 說明自行維護的注意事項，並告知未來隨時可加購

### 維護方案包含（摘要）

- 每月內容更新最多 2 次（文字 / 圖片小幅調整）
- 節日 / 活動 Banner 更換（每年最多 4 次）
- 網站正常運作監控（uptime）
- 基礎 SEO 維護
- 主機與 SSL 憑證管理
- 問題排除與技術支援

> 完整維護方案內容請參閱 [`maintenance-plan.md`](./maintenance-plan.md)

## 📌 常見問題處理

| 狀況 | 處理方式 |
|------|----------|
| 客戶遲遲不交素材 | Day 5 提醒 → Day 7 再次提醒 → Day 14 書面催告 → Day 30 合約自動終止條款啟動 |
| 客戶要求超出方案範圍 | 說明範圍，提供加購報價或升級方案建議 |
| 客戶不滿意初稿風格 | 了解具體不滿原因，在修改次數內盡量調整；大幅改版另行報價 |
| 客戶要求退款 | 依合約第九條處理（初稿前退 25%、初稿後不退） |
| 域名問題 | 協助客戶處理 DNS，必要時電話指導 |
