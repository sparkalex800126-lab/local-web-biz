# PageReady TW — Follow-up 節奏與回應腳本

> 最後更新：2026-03-21
> 原則：每次跟進都要帶新東西。沒新東西不要傳。

---

## 一、Follow-up 節奏總覽

| 天數 | 動作 | 管道 | 重點 |
|------|------|------|------|
| Day 0 | 初次訊息 + demo 連結 | LINE（優先）/ Email | 讓對方看 demo，不要推銷 |
| Day 3 | 第一次跟進 | 同管道 | 帶新資訊，不是「你看了嗎」 |
| Day 7 | 第二次跟進 | 同管道（或換管道） | 換角度切入 |
| Day 14 | 最後一次 + 退場 | 同管道 | 給台階下，保留關係 |

**4 次之後沒回覆 → 標記「暫緩」，2 個月後重新評估。**
**不要連續密集傳訊息，會變成騷擾。**

---

## 二、每一步的具體訊息模板

### Day 0：初次訊息

使用 `outreach-templates.md` 的 LINE 版本 A/B/C（依業種選）。

---

### Day 3：第一次跟進（沒回的情況）

> 重點：**帶一個新的東西**，不是重複上次的訊息。

#### 版本 3A —「截圖派」

```
{{owner_name}}，不好意思再打擾一下～

補充一個東西：我用手機打開那個 demo，截了一張圖給你看 👇
[附上手機截圖]

在手機上看起來蠻順的，客人搜到直接就能看到地址、電話、營業時間。

有空的話點開玩玩看，3 分鐘就看完了：
{{demo_url}}
```

**設計邏輯：**
- 截圖降低「點連結」的門檻（很多人懶得點）
- 具體描述網站能做什麼（地址、電話、營業時間）
- 「3 分鐘就看完」降低時間成本感知

#### 版本 3B —「幫忙觀察派」

```
{{owner_name}} 嗨～

上次傳的 demo 不知道你有沒有看到。沒關係，補充一個我觀察到的：

我搜「{{search_keyword}}」的時候，你們排在比較後面。
其實 {{store_name}} 的評價很好，有個網站的話 Google 排名會好蠻多的。

demo 連結再丟一次給你，有空看看：
{{demo_url}}
```

**設計邏輯：**
- 帶新資訊（搜尋排名觀察）
- 先誇再提問題，不會讓人不舒服
- 重新放連結，方便對方找

---

### Day 7：第二次跟進（換角度）

> 重點：如果前兩次都是講「網站長怎樣」，這次講「有網站之後會怎樣」。

#### 版本 7A —「同業案例派」

```
{{owner_name}}，最後分享一個東西～

最近幫竹北另一家 {{industry}} 做了網站，上線之後他們說 Google 搜尋來的客人明顯變多了。

你們的 demo 我還留著：
{{demo_url}}

如果有興趣想了解更多，隨時跟我說。
沒興趣的話也完全 OK，之後不會再打擾了 👋
```

**設計邏輯：**
- 用同業案例建立社會證明（sell outcomes）
- 預告這是倒數了，降低壓力
- 如果還沒有實際案例，可以改成：「最近研究發現，有自己網站的在地店家，客人搜到的機會平均多 3 倍」

#### 版本 7B —「換角度派」（從客人的角度切入）

```
{{owner_name}} 你好～

換個角度想：如果你是一個剛搬到竹北的新手爸媽，在 Google 搜「竹北 {{industry}}」——

看到有些店有自己的網站，有環境照片、理念介紹、收費方式⋯
有些只有 Google 地圖上一筆資料⋯

你會先聯繫哪一個？

{{store_name}} 的 demo 還在這裡：{{demo_url}}
有想法的話跟我說 🙂
```

**設計邏輯：**
- 讓老闆用客人的視角思考（perspective shift）
- 不說教，用問題引導
- 適合托嬰、月子中心這種重視信任的業種

---

### Day 14：最後一次 + 退場

> 重點：乾淨退場，不要黏。保留未來可能。

```
{{owner_name}}，

我猜你最近應該比較忙，完全理解！

那個 demo 我會保留一陣子，如果之後想看隨時跟我說。

祝 {{store_name}} 生意興隆 💪
```

**設計邏輯：**
- 只有 3 句話，超級短
- 「比較忙」給台階下（不是「你不理我」）
- 「保留一陣子」留個尾巴
- 祝福收尾，留下好印象

## 三、回應腳本 — 對方有回但說「要考慮」

### 情境 1：「我考慮一下」（最常見）

```
好的沒問題！不急～

你考慮的時候如果有任何問題，像是「這個可以改嗎」「費用大概多少」之類的，
隨時問我就好，我很樂意回答 😊
```

**然後 Day 5 後跟進一次：**

```
{{owner_name}} 嗨～想到什麼了嗎？

不管決定怎樣都沒關係，
如果 demo 有什麼想調整的地方，跟我說一聲我可以改給你看 👍
```

**設計邏輯：**
- 不施壓，但打開話題（讓對方知道可以問問題）
- 跟進時提供新的價值（「可以改給你看」）
- 提供免費修改 = 追加投入，提高對方的沉沒成本感知

---

### 情境 2：「我跟合夥人/另一半討論一下」

```
好啊，完全理解！如果需要的話，我可以把 demo 的重點整理成一頁，
方便你分享給 {{partner}}。

需要的話跟我說～
```

**設計邏輯：**
- 幫對方解決「怎麼跟別人說明」這件事
- 提供具體的工具（一頁重點整理）
- 讓轉介過程更容易

---

### 情境 3：「現在不太方便 / 之後再說」

```
沒問題！那我先不打擾了。

demo 會保留著，什麼時候方便了再跟我說就好。
祝一切順利 🙂
```

**然後在 pipeline 標記「暫緩」，6-8 週後可以用新角度重新接觸：**

```
{{owner_name}} 你好～好久不見！

之前那個 demo 我有更新了一些，現在又更好看了。
丟給你看看新版本：{{demo_url}}

如果時機比較合適了，再聊聊 🙂
```

---

### 情境 4：「我已經有網站了」

```
讚耶！那很好 👍

如果之後想改版或是覺得現在的網站需要調整，隨時找我。
竹北做網站的不多，我一直都在 😄
```

**設計邏輯：**
- 不要反駁、不要比較、不要貶低對方現有的網站
- 留下「之後想改版」的種子
- 「竹北做網站的不多」強化在地稀缺性

---

## 四、回應腳本 — 對方問價格

### 關鍵原則

1. **不要在 LINE 上直接丟一串價格表** → 數字沒有上下文會嚇到人
2. **先確認對方喜歡 demo 的程度** → 喜歡才報價
3. **報價時要連同價值一起講**

---

### 情境 1：直接問「多少錢？」

```
看你喜歡那個 demo 嗎？如果喜歡的話我跟你說～

我們有三個方案：

🅰 基本方案 $9,800
→ 就是 demo 那樣的網站，幫你上線 + 一年份主機

🅱 進階方案 $14,800
→ 多 Google 地圖優化 + 預約表單，讓客人更容易找到你、直接預約

🅲 完整方案 $19,800
→ 全部都有 + 幫你寫好所有文案內容 + 三個月的維護支援

一次付費，沒有月費。
如果覺得不值得，上線後 7 天內全額退費，沒問題的 👍

你覺得哪個比較適合 {{store_name}}？
```

**設計邏輯：**
- 先確認喜歡 → 再報價（Saraev：先確認需求再講價格）
- 每個方案一句話說明價值，不是列功能清單
- 「一次付費，沒有月費」消除最大的擔心
- 7 天退費 = guarantee → 降低風險感知
- 結尾用問題引導對話繼續

---

### 情境 2：「太貴了」/「預算不夠」

```
了解，預算的事可以聊～

你覺得哪個方案比較接近你要的？
我們可以看看怎麼調整，讓你用合理的預算先把網站上線。

畢竟早一天有網站，就早一天開始讓 Google 上的客人找到你 🙂
```

**不要做的事：**
- ❌ 不要立刻降價（會讓對方覺得原本是虛價）
- ❌ 不要說「這已經很便宜了」（對方的感受才是真的）
- ✅ 先了解對方的預算範圍，再看怎麼調整方案內容

---

### 情境 3：「我先問問別人的報價」

```
好啊，多比較是應該的！

如果有什麼問題或想討論，隨時找我。
不管最後選誰做，重要的是 {{store_name}} 能有一個好的網站 👍
```

**設計邏輯：**
- 不要表現出焦慮
- 自信但不自大
- 「不管選誰」展現大器 → 反而加分
- 信任你的 demo 已經展示了品質，讓作品替你說話

## 五、Pipeline 狀態管理

### 追蹤欄位

每個 prospect 記錄以下資訊：

```
店名：
聯絡人：
管道（LINE/Email）：
業種：
Day 0 日期：
使用模板版本：
目前狀態：🔵未聯繫 / 📤已寄出 / 📬已回覆 / 💬對話中 / ✅成交 / ❌拒絕 / ⏸️暫緩
下次動作：
下次動作日期：
備註：
```

### 狀態轉換規則

| 觸發事件 | 狀態轉換 | 下一步 |
|---------|---------|--------|
| 寄出首封 | 🔵 → 📤 | Day 3 跟進 |
| 對方回覆（任何回覆） | 📤 → 📬 | 依回覆內容選腳本 |
| 開始討論方案/價格 | 📬 → 💬 | 報價 + 跟進 |
| 4 次無回覆 | 📤 → ⏸️ | 2 個月後重新評估 |
| 明確拒絕 | → ❌ | 記錄原因，6 個月後可重新接觸 |
| 付款 | 💬 → ✅ | 開始執行 |

## 六、常見錯誤提醒

| ❌ 錯誤 | ✅ 正確做法 |
|---------|-----------|
| 「想確認你有沒有看到我的訊息」 | 帶一個新的觀察或截圖再傳 |
| Day 1 就傳三則訊息 | 遵守節奏，Day 0 → Day 3 → Day 7 → Day 14 |
| 對方說考慮就每天追問 | 等 5 天，帶新東西跟進 |
| 被拒絕就反駁 | 尊重，乾淨退場，留好印象 |
| 同一條街連續發五家 | 分散發送，避免店家互相比對 |
| 複製貼上沒改名字 | 每次都檢查變數有沒有換對 |
| 週末傳訊息 | 週二到週四的上午 10-12 點最佳 |
| 晚上 9 點之後傳 | 工作時間傳，尊重對方私人時間 |
